Policy Search

May 24, 2017 | 3 minutes read
Share this: Twitter Facebook Pinterest Google+

Consider a robot with states $x$ and inputs $u$. A policy $\pi(x,u)$ can be used to obtain the next best action $u_{t}=\pi(u_t|x_t)$ that maximizes some long term expected reward $R(x,u)$. The subscript $t$ refers to the time-step. A feedback controller also allows for getting the best $u_{t}$. The policy can also get the next state given the current state and the input $x_{t+1}=\pi(x_{t+1}|x_t,u_{t})$. This is unique to a policy and cannot be obtained from a feedback controller. Thus it is important to explore how a policy can obtained for accomplishing a task. A policy allows for planning future states. To list the advatages of a policy:

  • Can get $u_{t+1}$ and $x_{t+1}$
  • Can be used in a planner to plan a higher level task with the current policy.

A policy can be stochastic, denoted by $\pi(u|x)$ or deterministic ($u=\pi(x)$). In the stochastic setting, the control update is based on the probabilistic transition function $P(x_{t+1}|x_{t},u_{t})$. The goal is to obtain a trajectory $\tau=[x_0,u_0,….x_T,u_T]$ which maximizes the long term expected reward $R(\tau)$. The reward function gives an estimate of the performance of the policy. The reward function is mostly of the form $$R(\tau)=\sum_{t=0}^{T-1}r_t(x_t,u_t)+r_T(x_T)$$ where $r_T$ is the reward for the goal ( called as goal reward or final reward) and $r_t$ is the instatenous reward (or immediate reward). The immediate reward is usually on the control inputs to minimize large fluctuations (a regularizer) and the final reward is on achieving a particular task. In the infinite time setup, a discount factor $\rho$ is used to weigh future reward lower than the current reward to help with learning better policies at the beginning. The continuous domain case will be looked into later/never.

Value Function (bad for robotics)

A value function $V^\pi_t(x)$ is a measure of the quality of executing an action $u$ when in state $x$. This can be used to obtain a lookup table throughout the state space and can be used to compute a policy $\pi$ based on the value function mapping. However, robots have a large dimensions in their state space or/and not all states can achieved with random inputs. A classic example is a swingup of a pendulum where the top position is unreachable without a good swingup as the motor is incapable of sufficient torque. Value functions are generated iteratively by bootstrapping(?), making them biased.

Policy Search

Policy search explores the creation of these policies. Broadly, policy search can be split into two- model based and model free policy search. As their name suggests, model based methods attempt to fit the policy to an existing model which would be unlikely when the dimensionality of the system is large. Model free methods attempt to parametrize the relationship between the states and actions. Model free methods perform very well in learning single behaviors but lack generality.

Policy search methods use parameterized policies $\pi_\theta$, directly operating in the parameter space $\theta\in\Theta$
